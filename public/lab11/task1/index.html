Задание 1: Тестирование JWT Аутентификации

Инструкция по тестированию:

  1. Зарегистрируйте нового пользователя
  2. Выполните вход и получите токен
  3. Проверьте доступ к защищенным роутам (/profile, /admin)
  4. Протестируйте смену пароля
  5. Протестируйте выход (логаут)

Статус авторизации

Не авторизован

1. Регистрация пользователя

2. Вход (Login)

3. Профиль (защищенный роут)

4. Админский раздел (только для админов)

5. Смена пароля

6. Выход (Logout)

JWT Токен

Токен не получен

Проверьте токен на jwt.io

Результаты тестов из методички

Ниже представлены примеры запросов и ответов для тестов 1-5 из лабораторной работы:

Тест 1: Регистрация пользователя

Запрос:

POST http://localhost:3000/api/lab11/register
Content-Type: application/json

{
  "login": "user1",
  "password": "user1234"
}

Ответ (201):

{
  "id": 5,
  "login": "user1",
  "role_id": 1,
  "created_at": "2025-12-07T17:09:47.306Z"
}

Тест 2: Логин под этим пользователем

Запрос:

POST http://localhost:3000/api/lab11/login
Content-Type: application/json

{
  "login": "user1",
  "password": "user1234"
}

Ответ:

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NSwibG9naW4iOiJ1c2VyMSIsInJvbGUiOiJ1c2VyIiwiaWF0IjoxNzY1MTI3NTc4LCJleHAiOjE3NjUxMjg0Nzh9.pbdUVGyaqx6n_uDZYoza_M6E1vAsU4f3QqUp9dtnPjw",
  "user": {
    "id": 5,
    "login": "user1",
    "role": "user"
  }
}

Примечание: Скопируйте токен и проверьте его на jwt.io. В разделе Secret введите ваш SECRET_KEY из .env.local. В разделе Decoded Payload, Claims Table, строка Exp вы увидите время окончания срока действия токена (15 минут).

Тест 3: Запрос к /profile

Запрос:

GET http://localhost:3000/api/lab11/profile
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Ответ:

{
  "id": 5,
  "login": "user1",
  "role": "user"
}

Тест 4: Запрос к /admin (с токеном обычного пользователя)

Запрос:

GET http://localhost:3000/api/lab11/admin
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Ответ (403):

{
  "error": "Forbidden: admin only"
}

Тест 5: Изменение роли на admin и повторный запрос к /admin

SQL запрос для изменения роли:

UPDATE lab11.users 
SET role_id = (SELECT id FROM lab11.roles WHERE name = 'admin') 
WHERE login = 'user1';

Важно! После изменения роли необходимо заново выполнить логин и получить новый токен!

Запрос к /admin (с новым токеном):

GET http://localhost:3000/api/lab11/admin
Authorization: Bearer [новый токен после повторного логина]

Ответ:

{
  "message": "Добро пожаловать в админский раздел!",
  "user": {
    "id": 5,
    "login": "user1",
    "role": "admin"
  }
}

Скриншоты Postman

Для демонстрации работы API рекомендуется сделать скриншоты следующих запросов в Postman:

  1. Регистрация пользователя (POST /api/lab11/register)
  2. Логин (POST /api/lab11/login) - показать токен в ответе
  3. Профиль (GET /api/lab11/profile) - с заголовком Authorization
  4. Админ (GET /api/lab11/admin) - сначала с ролью user (403), затем с ролью admin (200)
